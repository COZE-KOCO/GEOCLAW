# 数据互通架构说明

## 概述

GEO优化工具平台的 **Web版** 和 **桌面版** 共享同一份数据，实现真正的数据互通。

## 架构图

```
┌─────────────────────────────────────────────────────────────────┐
│                         云端数据库                                │
│                    (Supabase PostgreSQL)                        │
└─────────────────────────────────────────────────────────────────┘
                              ▲
                              │
                              ▼
                    ┌─────────────────┐
                    │   后端 API      │
                    │  /api/accounts  │
                    └─────────────────┘
                              ▲
              ┌───────────────┴───────────────┐
              │                               │
              ▼                               ▼
    ┌─────────────────┐             ┌─────────────────┐
    │     Web 版      │             │    桌面版        │
    │  (OAuth授权)    │             │ (登录窗口提取)   │
    └─────────────────┘             └─────────────────┘
```

## 数据流程

### Web版账号绑定
```
用户点击平台图标
    ↓
跳转OAuth授权页面
    ↓
用户授权成功
    ↓
回调API保存账号到数据库
    ↓
刷新页面显示新账号
```

### 桌面版账号绑定
```
用户点击平台图标
    ↓
Electron打开登录窗口
    ↓
用户正常登录（扫码/账号密码）
    ↓
提取Cookie并调用API保存到数据库
    ↓
通知主窗口刷新显示新账号
```

### 数据同步
- **实时同步**：任何一端的操作都会立即同步到另一端
- **共享存储**：数据统一存储在云端数据库
- **无冲突**：基于businessId隔离，同一业务下的数据共享

## API接口

### 获取账号列表
```
GET /api/accounts?businessId={businessId}
```

### 创建账号
```
POST /api/accounts
{
  "action": "create",
  "data": {
    "businessId": "xxx",
    "platform": "wechat",
    "accountName": "微信公众号用户",
    "cookies": { ... }
  }
}
```

### 删除账号
```
POST /api/accounts
{
  "action": "delete",
  "data": {
    "id": "account-id"
  }
}
```

## 桌面版配置

### 开发模式
桌面版默认连接 `http://localhost:5000` API。

### 生产模式
需要配置环境变量 `API_BASE_URL` 指向服务器地址：

```bash
# macOS/Linux
export API_BASE_URL=https://your-domain.com

# Windows
set API_BASE_URL=https://your-domain.com

# 或在启动命令中指定
API_BASE_URL=https://your-domain.com electron .
```

## 优势

1. **数据统一**：无论在哪个平台操作，数据始终保持一致
2. **跨设备访问**：在办公室用桌面版绑定的账号，回家用Web版也能看到
3. **无需同步**：数据自动存储在云端，无需手动同步
4. **安全可靠**：数据库有备份和恢复机制

## 注意事项

1. **网络依赖**：桌面版需要网络连接才能同步数据
2. **登录状态**：两端需要使用相同的businessId才能看到相同的数据
3. **Cookie安全**：敏感凭证（Cookie）加密存储在数据库中
